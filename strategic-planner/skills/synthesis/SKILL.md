# Synthesis — Strategic Storyline

Build a storyline that directly answers the strategic question, organized by the logic of the answer — not by the process that produced it.

## Process

### Step 1: Identify the Governing Message
The governing message is the one-sentence answer to the strategic question. It must be:
- A clear recommendation or conclusion
- Supported by the evidence
- Specific enough to act on

### Step 2: Build the Headline Sequence
Organize the argument into 3-5 supporting headlines. Each headline:
- States a conceptual insight (the strategic "why")
- Is supported by evidence from research
- Connects logically to the next headline

**Structure test**: Read only the headlines in order. Does a coherent argument emerge?

### Step 3: Map Evidence to Headlines
For each headline, identify:
- Supporting data points (with source citations)
- Expert insights (if available)
- Contradicting evidence (must be addressed, not hidden)

### Step 4: Integrate Analysis Findings
The storyline must explicitly address:
- Top 2-3 pre-mortem failure modes (in a Risks/Counter-arguments section)
- MAP low-confidence dimensions (with honest uncertainty language)
- Reality check gaps (with explanation of why we expect to beat the base rate)

### Step 5: Confirm
Present the storyline to the user:
- Governing message
- Headline sequence
- Key evidence per headline
- Risks and counter-arguments

**Checkpoint**: User approves before report generation.

## Storyline Patterns

Choose the pattern that fits the question:

1. **Recommendation** — "We should do X because A, B, C"
2. **Assessment** — "The situation is X, which means Y"
3. **Comparison** — "Option A is stronger than B because X, despite Y"
4. **Prioritization** — "Focus on X first, then Y, because Z"
5. **Scenario** — "If X, then A; if Y, then B"

## Output

A confirmed storyline with governing message, headline sequence, evidence map, and integrated risk assessment.
