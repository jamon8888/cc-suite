# Product Pipeline Map

This map defines the skills associated with each phase of the product development lifecycle, as orchestrated by the `pipeline-orchestrator` skill.

---

### Phase 1: DISCOVER

**Goal:** Define and understand the problem and the user.

**Skills:**
- `proto-persona`
- `jobs-to-be-done`
- `discovery-interview-prep`
- `customer-journey-map`
- `problem-statement`
- `opportunity-solution-tree`
- `tam-sam-som-calculator`

**Output:** A validated Problem Statement and a clear definition of the target user.

---

### Phase 2: VALIDATE

**Goal:** Find the cheapest, fastest way to test the core assumptions behind the proposed solution.

**Skills:**
- `validation-checkpoint`
- `pol-probe-advisor`

**Output:** A go/no-go/pivot decision based on the results of the validation test.

---

### Phase 3: DESIGN

**Goal:** Translate the validated problem into a detailed product specification.

**Skills:**
- `prd-development`
- `roadmap-planning`
- `epic-breakdown`
- `user-story`
- `design-brief-generator`

**Output:** A complete Product Requirements Document (PRD) and a design brief for the prototype.

---

### Phase 4: PROTOTYPE

**Goal:** Create a tangible representation of the solution.

**Skills:**
- `figma-prototype`
- `stitch-loop`
- `remotion-video-generator`
- `prototype-to-video`

**Output:** An interactive prototype, a code-based mockup, or a product demo video.
